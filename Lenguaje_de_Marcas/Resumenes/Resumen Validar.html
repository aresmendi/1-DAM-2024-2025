Guía Avanzada de Validación de Formularios

(Con sintaxis de JavaScript tradicional)

El Formulario Extendido

Vamos a validar un formulario de registro más completo y realista, explicando cada parte.

1. Estructura y Estrategia Mejoradas

Mejoras de Accesibilidad (a11y)

Usamos el atributo aria-describedby para enlazar cada campo con su mensaje de error. Cuando un campo sea inválido, añadiremos aria-invalid="true", informando a los lectores de pantalla sobre el estado del campo.

Una Estructura de Código Sólida

Para gestionar el estado global del formulario, usamos un objeto que guarda si cada campo es válido o no. Esto simplifica mucho la lógica.

const estadoValidacion = {
    username: false,
    email: false,
    password: false,
    passwordConfirm: false,
    pais: false,
    terminos: false
};

2. Lógica de Validación Detallada

Cada campo tendrá su propia función de validación que actualizará el objeto estadoValidacion y el estado del botón de envío.

Validar Contraseñas Coincidentes

La validación de "Confirmar Contraseña" depende del valor del campo "Contraseña". Si el usuario cambia la contraseña original, debemos re-validar la confirmación.

function validarPasswordConfirm() {
    const passwordInput = document.getElementById('password');
    const confirmInput = document.getElementById('password-confirm');
    
    if (passwordInput.value !== confirmInput.value) {
        mostrarError(confirmInput, 'Las contraseñas no coinciden.');
        return false;
    }
    limpiarError(confirmInput);
    return true;
}

// Escuchador adicional: si cambia la contraseña, revalidamos la confirmación
document.getElementById('password').addEventListener('input', validarPasswordConfirm);

Validar Checkbox y Select

Usamos la propiedad .checked para el checkbox y comprobamos que el valor del select no sea el valor por defecto (vacío).

function validarTerminos() {
    const terminosCheckbox = document.getElementById('terminos');
    if (!terminosCheckbox.checked) {
        mostrarError(terminosCheckbox, 'Debes aceptar los términos.');
        return false;
    }
    limpiarError(terminosCheckbox);
    return true;
}

3. UX: Deshabilitar el Botón de Envío

El botón de envío estará deshabilitado hasta que todos los campos sean válidos. Una función se encarga de comprobar el estado global después de cada validación.

function actualizarEstadoBoton() {
    const submitButton = document.getElementById('submit-button');
    const esFormularioValido = Object.values(estadoValidacion).every(function(estado) {
        return estado === true;
    });
    
    submitButton.disabled = !esFormularioValido;
}
